# 食物卡路里分析模块（v1）

## 模块目标

支持两类输入并输出结构化营养结果：

- 文字输入：如“午饭吃了一碗牛肉面+一个鸡蛋”
- 图片输入：上传餐食图片

输出统一为：

- 食物项列表（名称、卡路里、蛋白/碳水/脂肪、份量提示、置信度）
- 总卡路里
- 汇总说明

## 输入模式

- `text-only`：仅文字描述
- `image-only`：仅图片
- `hybrid`：文字 + 图片联合分析

## 核心流程

1. 请求校验：至少提供文字或图片其中之一。
2. 文本识别：从自然语言中提取食物名称与份量提示。
3. 视觉识别：从图片中识别食物项与置信度。
4. 食物去重：按名称归一化去重（保留更高置信度项）。
5. 营养查询：基于 USDA 获取卡路里和三大营养素。
6. 结果组装：返回结构化结果与摘要。

## 接口定义

- 路径：`POST /usda/food/analyze`
- 请求体：

```json
{
  "text": "一碗牛肉面，一个水煮蛋",
  "imageDataUrl": "data:image/jpeg;base64,...",
  "userId": "user-001"
}
```

- 响应体（示例）：

```json
{
  "items": [
    {
      "name": "Beef noodle soup",
      "calories": 420,
      "proteinGrams": 20,
      "carbsGrams": 52,
      "fatGrams": 14,
      "confidence": 0.82,
      "servingHint": "1 bowl",
      "source": "usda"
    }
  ],
  "totalCalories": 420,
  "summary": "Detected 1 food items totaling 420 kcal."
}
```

## 准确性与风控

- 对低置信度项（<0.5）提示用户二次确认。
- 对未匹配到 USDA 的条目标记 `source=unknown`，避免伪精确。
- 当文字和图片冲突时，优先保留高置信度识别并提示用户确认。

## 迭代建议（下一步）

- 引入份量估计模型（餐盘大小、体积估算）。
- 增加中餐常见菜品本地映射库，提高命中率。
- 支持每条食物手动修正后回写，形成用户个性化校准。

# FitTrack AI Agent 产品需求文档（PRD v1）

## 1. 产品愿景

FitTrack AI Agent 是一款聚焦健身训练与饮食管理的个人健康智能体，核心目标：

- 提升训练质量与训练持续性。
- 提升饮食质量与长期可执行性。

产品应提供个性化、安全、可执行的建议，并将建议转化为每天可落地的行动。

## 2. 目标用户

- 需要明确计划与习惯引导的健身新手。
- 需要突破平台期的中级训练者。
- 需要稳定控热量的减脂用户。
- 需要系统增肌和渐进超负荷的增肌用户。
- 时间紧张、需要高效率方案的上班族。

## 3. 产品目标与 KPI

- 7 日留存：>= 35%
- 30 日留存：>= 20%
- 周训练完成率：>= 60%
- 每日饮食记录完成率：>= 50%
- 计划执行得分（饮食+训练）：>= 70/100

## 4. 核心需求模块

### 4.1 用户画像与基线评估

- 采集基础信息：性别、年龄、身高、体重、体脂估计、伤病情况、器械条件、饮食偏好、过敏信息。
- 采集时间约束：每周可训练天数、可训练时段、做饭条件。
- 生成基线指标：TDEE 估算、力量基线、动作熟练度。

验收标准：

- 新用户在 5 分钟内完成初始建档。
- 完成建档后可立即生成第一周计划。

### 4.2 目标管理

- 支持目标类型：减脂、增肌、重组、维持。
- 支持目标期限和每周推进节奏建议。
- 基于进展趋势与执行度自动调节目标节奏。

验收标准：

- 用户可随时修改目标。
- 当目标节奏过激时，Agent 能说明风险与取舍。

### 4.3 训练计划与训练陪练

- 生成周训练结构（家训/健身房双版本）。
- 生成每日训练内容（组数、次数、RPE、组间休息）。
- 根据伤病、器械、时长自动给出动作替代。
- 支持渐进超负荷逻辑（重量、次数、容量、减量周）。

验收标准：

- 用户可在聊天中按步骤完成当日训练。
- 训练后可根据反馈自动更新后续计划。

### 4.4 饮食计划与饮食记录

- 根据目标与画像生成热量与三大营养素目标。
- 提供每日饮食结构（3餐/4餐/轻断食等模式）。
- 支持文本与图片记录食物。
- 每条饮食记录返回营养估算与置信度。
- 超标/不足时给出下一餐纠偏建议。

验收标准：

- 单次饮食记录耗时 <= 20 秒。
- 每次记录后可实时刷新当日营养汇总。

### 4.5 每日打卡与自适应闭环

- 每日打卡维度：睡眠、精力、酸痛、饥饿、情绪、压力。
- 基于准备度动态调整训练强度与饮食目标。
- 识别不执行模式并自动切换策略。

验收标准：

- 每日打卡问题控制在 3-5 个关键问题。
- 同日输出 1 份明确行动清单。

### 4.6 安全与合规护栏

- 识别伤病风险、饮食失调信号、极端热量赤字等风险。
- 拦截不安全建议（医疗诊断、危险协议）。
- 在高风险场景输出“建议寻求专业帮助”模板。

验收标准：

- 不安全请求必须触发拒绝并提供安全替代方案。
- 护栏事件可审计、可追踪。

### 4.7 进展分析与反馈

- 看板指标：体重趋势、围度趋势、力量趋势、执行度趋势。
- 每周复盘：本周亮点、瓶颈、下周调整建议。
- 解释型建议：说明“为什么要这样调整”。

验收标准：

- 每 7 天自动生成周报。
- 每条建议包含原因与预期影响。

### 4.8 多模态交互

- 文本聊天为主入口。
- 图片上传用于食物识别。
- 语音输入/输出作为 v2 可选能力。

验收标准：

- 图片流程至少支持 JPG/PNG，低置信度时可回退到人工确认流程。

## 5. 非功能性需求

- P95 响应时延：
  - 纯文本 <= 3 秒
  - 图片识别 + 营养分析 <= 8 秒
- 可用性 >= 99.5%
- 可观测性：支持跨 Agent/Tool/Model 全链路追踪。
- 隐私与安全：
  - PII 静态加密
  - 用户级数据隔离
  - 管理操作审计日志

## 6. 分阶段范围

### Phase 1（MVP，6-8 周）

- 用户建档 + 目标管理 + 训练计划 + 文本饮食记录。
- 单编排 Agent + 2 个领域 Agent（营养/训练）。
- 周报复盘能力。

### Phase 2（增长，8-12 周）

- 图片食物识别与置信度评分。
- 每日打卡驱动的自适应调整闭环。
- 增强版进阶逻辑与平台期处理策略。

### Phase 3（高级能力）

- 可穿戴设备接入。
- 语音教练模式。
- 非诊断型健康风险预警。

## 7. 当前不纳入范围

- 医疗诊断或治疗建议。
- 实时紧急事件响应。
- 保险/临床流程集成。

## 8. 高风险假设

- 用户愿意持续进行饮食记录。
- 图片识别质量可支撑餐级决策。
- 初版提示词与护栏设计可有效降低不安全输出。

## 9. 验证计划

- 组织 20 人、2 周闭测。
- 重点观察留存、执行率、纠偏建议接受率。
- 按“对执行率与安全性的影响”优先排序迭代项。

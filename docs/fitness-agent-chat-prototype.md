# 健身 Agent 对话式原型规格

## 1. 概述

### 1.1 原型定位

本原型面向 `FitTrack.React` 现有 `/chat` 工作台，定义一版桌面优先的“综合健身教练”主界面。  
产品目标不是做通用聊天机器人，而是让用户通过一次持续对话，完成“当天计划、餐食分析、晚间复盘”三类高频任务。

### 1.2 目标用户

- 已有基本训练和饮食意识
- 会记录训练、体重、饮食，但缺少每天持续陪跑
- 需要一个能理解上下文、给出结构化建议、并帮助复盘的健身教练

### 1.3 页面边界

- 本轮只定义 `/chat` 主页面
- 不扩展登录、资料页、进度页的完整流程
- 默认用户已登录，且已有基础 `UserProfile`
- 默认沿用 `FitTrack.Copilot` 现有聊天线程、消息流、营养快照和资料接口

### 1.4 成功标准

用户进入页面后，应能在 10 秒内理解三件事：

1. 这是一个“可以持续陪跑”的综合健身教练
2. 我可以直接开始今天的训练/饮食对话
3. 重要结果会被沉淀成右侧卡片，而不只是停留在聊天气泡里

## 2. 页面结构

### 2.1 桌面线框

```text
+--------------------------------------------------------------------------------------------------+
| Coach                                                                                           |
|--------------------------------------------------------------------------------------------------|
| 左栏：线程列表              | 中栏：主对话区                                  | 右栏：洞察与结果 |
|-----------------------------|-------------------------------------------------|------------------|
| [New Today Session]         | [欢迎语 / 今日引导 / 推荐问题]                 | [Daily Summary]  |
| 今日计划 / 08:12            |                                                 |                  |
| 午餐分析 / 昨天             | 用户消息                                         | [Day Plan]       |
| 晚间复盘 / 周三             | Assistant 流式回复                               |                  |
|                             | Tool status                                      | [Meal Analysis]  |
|                             |                                                 |                  |
|                             | [文本输入框.........................] [图片] [发送] | [更多未来卡片位] |
+--------------------------------------------------------------------------------------------------+
```

### 2.2 布局原则

- 保持三栏工作台，不改成沉浸式单栏
- 中栏永远是视觉主轴，右栏作为“结果沉淀区”
- 左栏强调“连续性”，右栏强调“结构化结论”
- 首屏不是空输入框，而是引导式开场

## 3. 区块规格

### 3.1 左栏：线程列表

**作用**

- 承载每天、每餐、每次复盘的连续对话
- 让用户感受到“我不是每次都从零开始”

**默认内容**

- 顶部按钮：`新建今日会话`
- 下方按更新时间排序展示线程卡片
- 卡片信息包括：标题、最后更新时间、当前是否选中

**交互动作**

- 点击线程：切换当前对话上下文
- 点击 `新建今日会话`：创建新线程，并自动进入空消息状态

**显示规则**

- 始终显示
- 当没有任何线程时，只保留按钮和一条轻量提示：`今天还没有会话，先开始一次计划或复盘吧`

**与现有实现映射**

- 数据：`ConversationThread[]`
- 现有界面基础：`src/components/chat/chat-view.tsx`

### 3.2 中栏：欢迎开场区

**作用**

- 降低首次开聊门槛
- 让用户立刻知道这个 Agent 擅长什么
- 用已有 profile 做轻量个性化

**默认文案**

- 标题：`今天继续，我们把训练、饮食和恢复都安排清楚`
- 说明：`我会结合你的资料、历史会话和今天的输入，给出计划、分析和复盘建议。`
- 个性化副文案：
  - 若有 `displayName`：`{displayName}，要先安排今天的训练，还是先看吃什么？`
  - 若只有资料无昵称：`我已经记住了你的基础资料，可以直接开始今天的计划。`

**推荐问题**

- `帮我安排今天训练和饮食`
- `分析我这顿饭 / 这张照片`
- `帮我做今晚复盘`
- `今天适合高碳还是低碳？`

**交互动作**

- 点击推荐问题：自动填充输入框，并立即可发送
- 点击图片入口：直接选择餐食照片

**显示规则**

- 当线程存在但无消息时显示
- 一旦线程出现第一条消息，欢迎区折叠消失

**与现有实现映射**

- 数据来源：`UserProfile`
- 不依赖新接口，可直接基于现有 profile 信息生成默认文案

### 3.3 中栏：消息流

**作用**

- 展示用户与教练的连续上下文
- 承载解释、追问、流式生成和轻量工具反馈

**消息形态**

- 用户消息：右对齐，强调“我刚刚输入了什么”
- Assistant 消息：左对齐，支持 Markdown
- 流式回复：在消息尾部持续增量展示
- 工具状态：不单独做大卡片，只做消息区下方轻量过程文案

**默认行为**

- 新消息到来时自动滚到底部
- 结构化结果生成后，消息区只给摘要解释，不重复展示右栏大卡片全部内容

**显示规则**

- 有历史消息时始终显示
- 正在流式生成时显示临时 assistant 消息

**与现有实现映射**

- 数据：`ThreadDetail.messages`
- 类型：`ChatMessage`、`StreamEvent`
- 现有能力：支持 Markdown、流式 token、tool event、附件显示

### 3.4 中栏：输入区与快捷动作

**作用**

- 提供日常主输入入口
- 让“文本提问 + 餐食图片”成为同一级能力

**组件组成**

- 多行文本输入框
- 图片上传按钮
- 发送按钮
- 轻量工具状态文字
- 出错提示

**默认文案**

- 输入框 placeholder：`告诉我今天想练什么、吃了什么，或直接发一张餐食照片`

**交互动作**

- 文本发送：提交当前消息
- 图片上传：附着到当前输入上下文
- 图片已选择时：显示文件名或预览提示
- 发送中：输入区锁定，按钮进入 loading

**显示规则**

- 始终固定在中栏底部
- 报错后保留文本草稿和已选图片，避免用户重输

**与现有实现映射**

- 现有已有 `mealPhoto` 能力和上传预览提示
- 现有消息发送接口可继续复用，不要求新增 API

### 3.5 右栏：洞察与结果卡片

**作用**

- 沉淀本线程当前最重要的结构化结果
- 让用户无需翻聊天记录就能快速看结论

**卡片堆叠顺序**

1. `DailySummaryCard`
2. `DayPlanCard`
3. `MealAnalysisCard`

**排序原则**

- 默认把“今日全局状态”放最上面
- 计划类结果排第二，负责“下一步做什么”
- 餐食分析排第三，负责“这一餐怎么看”

**显示规则**

- 有对应结果时展示真实内容
- 无对应结果时允许显示弱化占位，但不能喧宾夺主
- 切换线程时，右栏卡片必须同步更新

**与现有实现映射**

- `DailySummaryCard`
- `DayPlanCard`
- `MealAnalysisCard`
- `NutritionSnapshot`

## 4. 关键状态

### 4.1 首次进入空状态

**触发条件**

- 新线程刚创建
- 当前线程暂无消息

**界面要求**

- 中栏显示欢迎语、推荐问题、图片入口
- 左栏保留线程结构
- 右栏显示弱化占位，提示“今天的总结和计划会出现在这里”

### 4.2 对话进行中

**触发条件**

- 用户已发送消息，Assistant 正在流式返回

**界面要求**

- 中栏显示用户消息
- Assistant 消息逐字流式增长
- 底部显示轻量工具过程文案，例如：`正在分析餐食照片...`
- 输入区进入 loading / locked 状态

### 4.3 生成日计划

**触发条件**

- 用户提出“今天怎么练 / 怎么吃 / 帮我安排今天”

**界面要求**

- 对话流中给出简洁解释和执行理由
- 右栏置顶显示 `DayPlanCard` 风格卡片
- 卡片至少包含：
  - 日类型
  - 热量/蛋白/碳水/脂肪目标
  - 2 到 3 条执行 guidance

### 4.4 生成餐食分析

**触发条件**

- 用户发送餐食图片
- 用户询问某一餐是否合适

**界面要求**

- 输入区或消息区显示已附着图片提示
- 流式阶段显示“正在分析餐食”过程状态
- 右栏显示 `MealAnalysisCard`
- 卡片至少包含：
  - 餐次名称
  - 宏量摘要
  - 评分
  - 2 到 3 条调整建议

### 4.5 生成今日总结

**触发条件**

- 用户完成一次综合对话
- 或者用户明确请求复盘

**界面要求**

- 右栏显示 `DailySummaryCard`
- 卡片至少包含：
  - 今日摄入或 readiness 指标
  - 体重/恢复相关摘要
  - 一句当天 callout
  - 可选的 1 到 3 条后续动作

### 4.6 无历史线程但已有 profile

**触发条件**

- 用户刚进入产品，profile 已存在，但还没有任何聊天历史

**界面要求**

- 欢迎区文案应体现“我知道你是谁，但还没开始今天的会话”
- 示例文案：`我已经读取了你的基础资料，先帮你安排今天训练，还是先分析当前饮食？`

### 4.7 失败与降级状态

**触发条件**

- 流式接口中断
- 返回 error event
- 当前线程加载失败

**界面要求**

- 在输入区上方显示清晰错误信息
- 保留用户草稿和已附着图片
- 提供明确重试方式：
  - 重新发送当前消息
  - 重新加载当前线程
- 不清空右栏已有结果，除非当前线程切换

## 5. 对话体验规则

### 5.1 首屏规则

- 首屏必须是引导式开场
- 不允许出现“空白中栏 + 一个孤立输入框”的通用聊天布局

### 5.2 推荐问题规则

- 推荐问题围绕当天闭环，不围绕泛知识问答
- 优先顺序：
  1. 今日计划
  2. 餐食分析
  3. 晚间复盘

### 5.3 结果呈现规则

- 重要结论必须双通道输出：
  - 对话中给解释
  - 右栏给结构化沉淀
- 不允许把“今日计划”只写在消息气泡里

### 5.4 工具状态规则

- 工具状态只做轻量文本反馈
- 不做大面积进度条或覆盖层
- 推荐文案风格：
  - `正在整理今天的宏量目标...`
  - `正在分析这张餐食照片...`
  - `正在汇总今天的恢复和摄入情况...`

## 6. 与现有实现的映射

### 6.1 可直接复用

- 聊天主工作台：`src/components/chat/chat-view.tsx`
- 页面入口：`src/app/chat/page.tsx`
- 资料信息：`src/app/settings/profile/page.tsx`
- 结果卡片：
  - `DailySummaryCard`
  - `DayPlanCard`
  - `MealAnalysisCard`

### 6.2 直接对应的数据结构

- 线程列表：`ConversationThread`
- 当前会话详情：`ThreadDetail`
- 消息流：`ChatMessage`
- 流式事件：`StreamEvent`
- 营养快照：`NutritionSnapshot`
- 用户基础信息：`UserProfile`

### 6.3 本轮不要求新增的能力

- 不新增后端接口
- 不新增新的线程模型
- 不重新设计 Agent 编排
- 不定义训练周期、穿戴设备、日历同步等扩展能力

## 7. 未来增强项

以下内容允许在原型中以视觉占位方式出现，但不属于本轮必需范围：

- 周训练负荷卡片
- 连续打卡 streak
- 恢复评分和睡眠来源接入
- 动作指导卡片
- 周/月趋势复盘卡片

若后续需要这些卡片，应单独评估接口和数据结构，而不是假定当前聊天接口已经支持。

## 8. 验收场景

### 场景 1：首次进入聊天页

- 用户进入 `/chat`
- 能立即看到欢迎语和推荐问题
- 能通过一个明显入口开始“今日计划”对话

### 场景 2：上传餐食照片

- 用户附加一张餐食图片
- 页面显示图片已附着提示
- 发送后看到分析中状态
- 完成后右栏出现餐食分析卡片

### 场景 3：完成一轮综合提问

- 用户询问当天训练和饮食安排
- Assistant 在中栏给出解释
- 右栏出现日计划卡片和今日总结卡片

### 场景 4：切换历史线程

- 用户点击左栏历史线程
- 中栏消息切换
- 右栏卡片同步刷新到该线程对应结果

### 场景 5：发生异常

- 流式返回失败或线程加载失败
- 页面给出明确错误文案
- 草稿内容未丢失
- 用户可以重试

## 9. 实现前检查清单

- 每个关键状态都有独立说明
- 每个卡片都定义了触发条件和显示位置
- 每个推荐问题都有明确作用
- 每个主要区块都能映射到现有组件或类型
- 所有未来增强项都已与本轮必需范围分开
